This page show how we should always submit forms : with the Post-Redirect-Get pattern and with server validation. This is the common way to do. The form has been loaded with a new ContactUsModel with redirect attributes :

  • successRedirect : define on which page we are redirected if the form is successfully submited.
  • failRedirect : define on which page we are redirected otherwise.

For this example, we are always redirected on this page, with errors or not. Some will say that it is not a good idea to put redirect attribute inside FormModel : I disagree, for many reasons :

  • Consistency : Since we want complete validation with messages and use annotation validation, it is much more better to create a specific FormModel object and not use entity object (the validation could be contextual and moreover, use entity as model often brings problems, especially in transactionnal context). So, since we have a specific form model, why not fully exploit it ?
  • Reusability : this allow us to have a specific and independant form controller, which is only linked to the form fragment, not the page which use this fragment !
