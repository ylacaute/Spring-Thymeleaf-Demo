To have a better use experience, we can add HTML5 validation. The form below is the same as in the classic demo page, the only difference is that we had new HTML/JS/Thymeleaf stuffs in the fragment.

  • required : inputs which can't be empty
  • pattern : inputs validated by a regular expression
  • onChange : reset HTML5 validation
  • th:onInvalid : call a validation JS method with I18N key message as arguments

Note : there are still some limitations, it is still not possible to check an input text field on its min or max length in a native html5 validation way. To do so we must use a pattern (regexp) and forget the idea to have two differents messages for size and pattern matching. Another way could be to hack the form validation. In this example, we let the server check the lengths.
